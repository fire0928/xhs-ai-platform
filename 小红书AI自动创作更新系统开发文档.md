# 小红书AI自动创作更新系统开发文档

# 1\. 文档概述

## 1\.1 文档目的

本文档用于明确小红书AI自动创作更新系统的开发需求、架构设计、功能实现、技术选型及测试部署规范，为开发人员提供开发依据，为产品、测试人员提供需求参考，确保系统开发符合用户预期，保障开发过程有序推进、各模块协同兼容。

## 1\.2 系统概述

本系统是一套基于C/S架构的小红书AI自动创作、自动发布系统，支持电脑端、鸿蒙手机端、小程序端多端适配。核心功能涵盖用户注册登录、小红书账号绑定、账号数据统计分析、AI图文创作（多Agent可选）、图文审核、发布队列管理、自动发布，以及服务器端前台/后台的多维度管理功能。系统核心目标是降低用户小红书运营成本，实现内容自动生成、批量管理、定时发布，同时通过服务器端实现对所有用户、AI模型、内容的集中管控。

## 适用范围

本文档适用于本系统的开发人员、测试人员、产品人员、运维人员及相关管理人员，作为系统开发、测试、部署、维护的核心依据。

# 2\. 需求分析

## 2\.1 核心需求

1. 架构需求：采用C/S架构，用户数据（账户信息、小红书账号信息、操作记录等）统一存储在服务器，为每个用户建立独立数据档案，确保数据隔离与安全。

2. 多端适配：支持电脑端、鸿蒙手机端、小程序端，三端数据实时同步，功能一致（除服务器后台仅支持电脑端外）。

3. 用户管理：用户通过手机号完成首次注册，登录后可绑定1个或多个小红书账号，绑定后可获取账号相关信息并同步至服务器。

4. 数据统计分析：登录界面展示绑定小红书账号当日数据（浏览量、点赞量、收藏量等），数据分析页面支持查询过往历史数据，生成基础数据报表。

5. AI创作模块：基于大模型API实现图文内容自动生成，服务器后台可配置大模型API及多个Agent，用户端可自主选择Agent；用户输入创作方案或标题方向，可选择生成1\-10条内容，生成流程为先出标题，用户确认/修改后，再生成对应文案及配图。

6. 图文审核模块：生成的图文以朋友圈式逐条展示，用户可逐条查看、审核，审核通过后可加入发布队列，并设置具体发布时间。

7. 自动发布：系统根据发布队列中的内容及设定的发布时间，自动完成小红书内容发布，无需用户手动操作。

8. 服务器端管理：分为前台（用户个人管理）和后台（超级管理员管理），前台支持用户自身的内容管理、数据分析；后台支持所有用户管理、AI大模型管理、全量内容管理、分用户内容及数据分析。

## 非核心需求

1. 安全性需求：用户密码加密存储，小红书账号信息加密保存，防止信息泄露；接口请求需进行身份验证，杜绝非法访问。

2. 可扩展性需求：系统架构支持后续新增端（如iOS端、Android端），支持大模型API、Agent的灵活新增与删除，支持功能模块的迭代升级。

3. 易用性需求：界面设计简洁直观，操作流程清晰，降低用户学习成本；AI生成内容支持手动修改，发布队列支持编辑、删除、调整顺序。

4. 稳定性需求：系统需保证7\*24小时稳定运行，自动发布功能不出现漏发、错发，数据统计准确无误，多端同步无延迟。

## 约束条件

1. 技术约束：手机端仅开发鸿蒙系统版本及小程序版本，暂不支持iOS、Android原生版本；服务器端后台仅支持电脑端访问。

2. 接口约束：需对接小红书官方接口（或合规第三方接口），用于账号绑定、数据获取、内容发布，确保接口调用合规，避免账号风险。

3. 性能约束：AI生成内容响应时间≤10秒/条，批量生成10条内容响应时间≤60秒；多端同步延迟≤30秒；自动发布成功率≥99%。

4. 存储约束：服务器需满足用户数据、图文内容（图片、文案）的长期存储，支持至少1000名用户的并发使用，单用户可存储至少1000条生成内容及发布记录。

# 3\. 系统架构设计

## 3\.1 整体架构（C/S架构）

本系统采用C/S（客户端\-服务器）架构，分为客户端层、服务器层、数据存储层三层，各层独立运行、协同交互，确保系统灵活性与可维护性。

### 客户端层

作为用户直接操作的入口，分为三个终端，三端共用一套业务逻辑，数据通过接口与服务器实时同步，确保操作一致性。

1. 电脑端：支持所有用户功能（注册登录、账号绑定、AI创作、审核发布、数据分析）及服务器前台/后台管理功能，适配Windows、MacOS系统。

2. 鸿蒙手机端：支持用户核心功能（注册登录、账号绑定、AI创作、审核发布、数据分析），适配鸿蒙2\.0及以上版本手机，支持离线缓存未发布内容。

3. 小程序端：支持用户核心功能，适配微信小程序，无需安装客户端，通过微信搜索即可访问，满足临时操作需求，支持微信授权快捷登录。

### 服务器层

作为系统核心，负责业务逻辑处理、接口提供、数据校验、AI模型调用、自动发布调度等，分为前台服务和后台服务。

1. 前台服务：对应用户个人管理界面，提供用户相关业务接口（注册登录、账号绑定、数据查询、AI创作、审核发布等），处理用户的各类操作请求。

2. 后台服务：对应超级管理员管理界面，提供全局管理接口（用户管理、AI大模型管理、Agent配置、全量内容管理、分用户数据分析等），负责系统全局配置与管控。

### 数据存储层

负责所有数据的存储与管理，采用分布式存储方案，确保数据安全、可扩展，分为用户数据、业务数据、系统数据三类。

## 技术选型

### 客户端技术选型

|终端类型|核心技术|辅助技术|
|---|---|---|
|电脑端|Electron（跨平台桌面应用开发）|Vue3、Element Plus、Axios（接口请求）|
|鸿蒙手机端|HarmonyOS SDK、ArkTS（鸿蒙开发语言）|鸿蒙UI组件库、Retrofit（接口请求）|
|小程序端|微信小程序原生框架、UniApp（跨端适配）|WeUI组件库、Axios、微信授权API|

### 服务器端技术选型

|模块|核心技术|说明|
|---|---|---|
|后端框架|Spring Boot 3\.x|快速开发、轻量级、支持分布式部署|
|接口开发|Spring MVC、Swagger|规范接口开发，提供接口文档，便于调试|
|AI模型调用|OkHttp、自定义API封装|支持多大模型API配置，灵活调用不同Agent|
|定时任务（自动发布）|Quartz、Spring Scheduler|精准调度发布队列，确保按时发布|
|身份验证|JWT（JSON Web Token）、微信小程序授权|无状态验证，支持多端同时登录，支持小程序微信快捷授权，确保接口安全|

### 数据存储技术选型

|数据类型|存储技术|说明|
|---|---|---|
|结构化数据（用户信息、账号信息、发布记录等）|MySQL 8\.0（主从复制）|稳定可靠，支持事务，主从复制确保数据备份|
|非结构化数据（图片、文案草稿等）|MinIO（分布式对象存储）|高可用、高扩展，适合存储大量图片等文件|
|缓存数据（高频访问数据、用户登录状态等）|Redis|提升访问速度，减少数据库压力|

### 其他技术

1. 数据加密：MD5（密码加密）、AES（敏感信息加密，如小红书账号信息）

2. 日志管理：Logback，记录系统操作日志、错误日志，便于排查问题

3. 部署技术：Docker容器化部署，Nginx反向代理，支持负载均衡

## 接口设计原则

1. 统一性：所有接口采用RESTful风格，URL命名规范、参数格式统一，便于多端调用。

2. 安全性：所有接口需携带JWT令牌或小程序授权凭证进行身份验证，敏感接口（如AI模型配置、用户删除）需额外权限校验。

3. 可扩展性：接口设计预留扩展字段，支持后续功能迭代，避免接口频繁变更。

4. 容错性：接口需处理异常情况（如参数错误、接口调用失败），返回明确的错误信息，便于客户端处理。

# 4\. 详细功能设计

## 4\.1 用户模块（全端通用）

### 注册功能

1. 触发条件：用户首次打开客户端，未登录状态下点击“注册”按钮。

2. 操作流程：输入手机号 → 获取验证码（短信验证，6位数字，有效时间5分钟） → 输入验证码 → 设置登录密码（6\-16位，包含字母\+数字） → 确认注册。

3. 业务逻辑：服务器校验手机号唯一性（不可重复注册）、验证码正确性，校验通过后，加密存储用户密码，创建用户独立数据档案，返回注册成功信息，自动跳转至登录界面。

4. 异常处理：手机号已注册、验证码错误/过期、密码格式不符合要求，返回对应错误提示，引导用户修正。

### 登录功能

1. 操作流程：输入手机号 → 输入登录密码 → 点击“登录”（支持“记住密码”功能，默认记住7天）；小程序端额外支持微信授权快捷登录，授权后绑定手机号即可完成登录。

2. 业务逻辑：服务器校验手机号、密码正确性（或小程序微信授权有效性），校验通过后生成JWT令牌，返回给客户端（客户端存储令牌，用于后续接口请求），同时查询用户绑定的小红书账号信息及当日数据，跳转至首页。

3. 辅助功能：忘记密码 → 输入手机号 → 获取验证码 → 重置密码 → 重新登录。

### 小红书账号绑定

1. 操作入口：登录后，首页点击“绑定小红书账号”，或进入“我的”页面进行绑定。

2. 绑定流程：输入小红书账号（手机号/用户名） → 输入小红书密码 → 点击“绑定”（对接小红书接口，验证账号有效性）。

3. 业务逻辑：验证通过后，加密存储小红书账号信息至服务器（与当前用户绑定，一对一或一对多绑定，最多绑定3个），同步获取该账号的基础信息（头像、昵称、粉丝数）及当日数据，展示在登录首页。

4. 后续操作：支持解绑小红书账号（解绑后，该账号相关数据不再同步，历史数据保留）、切换绑定的小红书账号（切换后，首页展示对应账号的数据）。

## 4\.2 数据分析模块（全端通用）

### 首页数据展示

用户登录后，首页默认展示当前绑定小红书账号的当日数据，包含以下内容：

1. 核心数据：当日浏览量、点赞量、收藏量、评论量、新增粉丝数。

2. 数据趋势：当日数据与昨日同期数据对比（上升/下降百分比，直观展示）。

3. 快捷入口：点击“查看详情”，跳转至数据分析页面。

### 数据分析页面

1. 数据筛选：支持按时间筛选（今日、昨日、近7天、近30天、自定义时间范围），支持按绑定的小红书账号筛选（多账号绑定情况下）。

2. 数据展示：
        

    - 趋势图表：以折线图/柱状图展示核心数据（浏览、点赞等）的变化趋势，支持鼠标悬浮（电脑端）、点击（鸿蒙端、小程序端）查看具体数值。

    - 明细数据：展示每日核心数据明细，支持导出Excel文件（电脑端专属功能）。

    - 内容分析：展示已发布内容的单条数据（每条内容的浏览、点赞、收藏等），便于用户分析优质内容特征。

3. 业务逻辑：服务器从小红书接口同步历史数据，结合本地存储的发布记录，生成数据分析报表，实时更新数据。

## 4\.3 AI创作模块（全端通用）

### Agent选择

1. 操作入口：进入“创作”模块，顶部显示“Agent选择”下拉菜单。

2. 功能说明：服务器后台已配置多个Agent（每个Agent对应不同的创作风格，如美妆、穿搭、美食、知识科普等），用户可根据自身小红书账号定位，选择对应的Agent（默认选择第一个Agent）。

3. 业务逻辑：用户选择Agent后，服务器记录用户选择偏好，后续生成内容时，调用该Agent对应的大模型配置。

### 内容生成设置

1. 创作输入：用户输入创作方案（如“3条美妆教程，主打平价粉底液”）或标题方向（如“平价粉底液推荐”），支持输入关键词、风格要求（如“温柔风、干货向”）。

2. 生成条数选择：下拉菜单选择生成条数（1\-10条，默认1条）。

3. 提交生成：点击“生成标题”按钮，触发AI生成逻辑。

### 标题生成与确认

1. 标题生成：服务器根据用户输入的创作需求、选择的Agent，调用对应大模型API，生成对应条数的标题（每条内容1个标题）。

2. 标题编辑：用户逐条查看标题，可对标题进行修改（直接编辑）、替换（点击“重新生成该标题”），确认所有标题无误后，点击“确认标题，生成文案配图”。

3. 异常处理：标题生成失败（如API调用异常），返回错误提示，支持重新提交生成。

### 文案与配图生成

1. 生成逻辑：服务器根据确认后的标题、用户创作需求、Agent风格，调用大模型API生成对应文案（符合小红书风格，包含表情、话题标签\#），同时调用图片生成API（或对接第三方图片库），生成与文案、标题匹配的配图（每张文案对应1\-3张配图，默认1张）。

2. 内容编辑：生成完成后，用户可对文案进行修改（编辑文字、添加/删除话题标签），对配图进行替换（重新生成该配图、上传本地图片；小程序端支持微信相册上传）。

3. 保存功能：支持将生成的图文内容保存为草稿，后续可继续编辑、审核。

## 4\.4 图文审核模块（全端通用）

1. 审核展示：生成的图文内容以朋友圈式逐条展示，每条内容包含标题、文案、配图，底部显示“审核通过”“重新编辑”“删除”按钮。

2. 审核操作：
        

    - 审核通过：点击“审核通过”，弹出发布时间选择框（支持立即发布、定时发布，定时发布可选择具体日期、时间），确认后加入发布队列。

    - 重新编辑：点击“重新编辑”，跳转至编辑页面，修改标题、文案、配图后，重新提交审核。

    - 删除：点击“删除”，确认后删除该条图文内容（草稿可恢复，已删除的非草稿内容不可恢复）。

3. 批量操作：电脑端支持批量选择多条图文，批量审核通过、批量删除（鸿蒙手机端、小程序端暂不支持批量操作）。

## 4\.5 发布队列与自动发布模块（全端通用）

### 发布队列管理

1. 队列展示：进入“发布队列”页面，按发布时间排序（从近到远），展示每条内容的标题、配图、发布时间、绑定的小红书账号、发布状态（待发布、发布中、已发布、发布失败）。

2. 队列操作：
        

    - 编辑：可修改待发布内容的发布时间、绑定的小红书账号（需未到发布时间）。

    - 删除：可删除待发布内容（已发布、发布中的内容不可删除）。

    - 调整顺序：电脑端支持拖拽调整待发布内容的顺序，鸿蒙手机端、小程序端支持上下移动调整顺序。

    - 手动发布：对待发布内容，可点击“立即发布”，跳过定时设置，直接发布。

### 自动发布

1. 调度逻辑：服务器通过定时任务（Quartz），实时检测发布队列中的待发布内容，当到达设定的发布时间时，自动调用小红书发布接口，完成内容发布。

2. 状态更新：发布成功后，更新内容发布状态为“已发布”，同步记录发布时间、发布结果；发布失败（如接口异常、账号异常），更新状态为“发布失败”，返回失败原因，支持点击“重新发布”。

3. 异常处理：发布过程中出现网络中断、接口超时，系统自动重试2次（间隔5分钟），仍失败则标记为发布失败，通知用户（客户端消息提示、小程序服务通知）。

## 4\.6 服务器端前台（用户个人管理，仅电脑端）

服务器端前台为用户个人管理界面，除包含客户端所有用户功能外，新增以下核心管理功能：

1. 内容管理：查看个人所有生成的图文内容（草稿、审核中、已审核、已发布、已删除），支持按状态、时间筛选，支持批量编辑、批量删除、批量导出。

2. 数据分析：比客户端新增“内容效果分析”，可查看单条内容的全周期数据（发布后7天、30天的浏览、点赞等变化），生成内容效果报表，辅助用户优化创作方向。

3. 个人设置：修改登录密码、绑定手机号，管理小红书账号绑定（解绑、新增绑定），设置AI创作默认参数（默认Agent、默认生成条数）。

## 4\.7 服务器端后台（超级管理员管理，仅电脑端）

服务器端后台为超级管理员专属界面，负责系统全局管理，核心功能分为四大模块：

### 用户管理

1. 用户列表：展示所有注册用户的信息（用户ID、手机号、注册时间、绑定小红书账号数量、账号状态（正常、禁用）、注册终端（电脑端/鸿蒙端/小程序端））。
